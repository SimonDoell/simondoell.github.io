Real Time Liquid Particle Simulation using C++ and SFML

Description:

The number of particles is controlled via a variable. Each particle is stored in a list, and a loop updates their acceleration based on their proximity to other particles.

The simulation runs on a single CPU core, which limits performance—a common constraint in C++ programs.

Particles that are too close repel each other, while those farther apart exert a slight attraction. This interaction creates semi-realistic wave-like motion that also reflects off the boundaries.

SFML is used only for rendering and movement, it does not handle any simulation logic or calculations.

Particle brightness increases with speed.

Adaptive Real Time Gravity Simulation using C++ and SFML

Description:

This is a 2D gravity simulation that uses two separate lists: one for objects that emit gravitational forces (displayed in blue), and another for objects that are affected by gravity (displayed in white).

Performance is limited by single-core processing, but the simulation can handle up to 15,000 objects smoothly.

Gravity is calculated using the true inverse-square law for realistic force simulation.

Objects are removed from the simulation when they collide with a gravity-emitting object.

Objects influenced by gravity do not interact with each other in any way.
